# CRT 复古电视特效 · 从 0 到 1 实战总结

> 用 Three.js + 自定义 GLSL 着色器，在网页 Hero 区域实现照片融入老电视的复古噪点特效。完整记录设计思路、迭代过程和核心参数调优。

---

## 1. 起点：纯动态噪点渐变

最初的 Hero 区域是一个 WebGL Canvas，用 GLSL 着色器实时生成**双色渐变 + 噪点扰动**：

```glsl
// 核心：sin 伪随机噪点 + 距离场渐变
float noise(vec2 p){return fract(sin(dot(p,vec2(127.1,311.7)))*43758.5453);}
float dist=distance(uv, vec2(0.5+sin(uTime*0.3)*0.2, 0.5+cos(uTime*0.4)*0.2));
float gradient=smoothstep(0.0, 1.2, dist+n);
vec3 col=mix(uColor1, uColor2, gradient);
```

效果像一个复古电视机的雪花屏，光晕随时间的 sin/cos 缓慢移动。**没有图片，纯程序化生成。**

## 2. 叠加照片：着色器内纹理融合（Option B）

用户想在保留动态效果的同时融入照片。选择了**着色器内混**方案——把照片作为 `sampler2D` 纹理传入 GLSL，在片元着色器内与噪点合成。

### 加载纹理

```javascript
const loader = new THREE.TextureLoader();
const photoTex = loader.load('hero-photo.png');
// 传入 uniform
uniforms: { uPhoto: { value: photoTex } }
```

图片文件名 `WebGL 复古电视特效.PNG` 在 Web 上加载失败（URL 编码问题），改为 `hero-photo.png` 解决。

## 3. 三次迭代调优

### 第一版：致命题——特效太重

```glsl
float grain = noise(uv*80.0+uTime*0.5)*0.12-0.06;
float scanline = sin(uv.y*800.0+uTime*0.2)*0.04;
col = photo.rgb + grain + scanline;  // 直接在照片上叠加
```

**问题**：噪点、扫描线、色散直接覆盖照片，照片看不清。用户反馈"特效罩在图片上面"。

### 第二版：CRT 融合思路转变

**核心认知**：不要"给照片加滤镜"，而是**"让照片在 CRT 屏幕里播放"**。

```glsl
// 照片是主体，特效融入：
vec3 col = photo.rgb;
col *= scanline;   // 扫描线乘上去（不是加）
col *= maskP;       // 磷光点阵
col += grain;       // 颗粒微量叠加
col += sheen;       // 屏幕反光
```

关键改变：
- 噪点从**加法叠加**变为**乘法融入**
- 扫描线密度降低，强度减小
- 加入 CRT 桶形畸变（屏幕曲面）
- 加入荧光辉光、暖色温

```glsl
// CRT 桶形畸变
vec2 centered = uv - 0.5;
float barrel = 1.0 + dot(centered,centered) * 0.08;
vec2 uvCRT = 0.5 + centered * barrel;
```

### 第三版：加强抖动 + 降速

用户反馈"躁动特效猛烈一点"→"让特效变慢一点"。发现**强度和速度是独立维度**：

| 调整 | 参数 | 策略 |
|------|------|------|
| 水平抖动 | `noise()*0.012 + noise()*0.006` | **双层噪点叠加**，不同频率 |
| 垂直抖帧 | `noise(uTime)*0.006` | V-hold 模拟 |
| 撕裂带 | `step(0.92, noise())` | 随机阈值触发横条偏移 |
| 颗粒 | `noise()*0.12 + noise()*0.06` | 双层颗粒 |
| 色彩闪断 | `step(0.94, noise())` | 条件触发 RGB 通道错位 |
| 速度 | 所有 `uTime` 系数 **÷3** | 保持强度，放慢节奏 |

## 4. 最终着色器架构

```
┌──────────────────────────────────────────┐
│  1. UV 输入                               │
│  2. CRT 桶形畸变（barrel distortion）       │
│  3. 信号抖动（水平 + 垂直 + 撕裂带）         │
│  4. 采样照片纹理                            │
│  5. 色差分离（边缘渐强）                     │
│  6. 荧光辉光（亮度阈值）                     │
│  7. 扫描线 × 磷光点阵（乘法融入）            │
│  8. 颗粒 + 反光 + 暗角（加法/混合）          │
│  9. 暖色温 + 亮度闪烁 + 色彩闪断            │
│ 10. 屏幕边框阴影                           │
└──────────────────────────────────────────┘
```

## 5. 关键参数速查表

### 噪点函数
```glsl
float noise(vec2 p) {
  return fract(sin(dot(p, vec2(127.1, 311.7))) * 43758.5453);
}
```
标准伪随机函数，输入 UV×频率 + 时间偏移，输出 0~1 灰度噪点。

### 效果强度参数（可独立调节）

| 参数 | 代码位置 | 当前值 | 调大效果 |
|------|----------|--------|----------|
| 桶形畸变 | `barrel = 1.0 + r2*0.08` | `0.08` | 屏幕更凸 |
| 水平抖动 | `noise()*0.012 + noise()*0.006` | `0.012, 0.006` | 更抖 |
| 垂直抖动 | `noise()*0.006` | `0.006` | 跳帧更强 |
| 撕裂带频率 | `step(0.92, noise())` | `0.92` | 降低→更多撕裂 |
| 撕裂带强度 | `noise()*0.04` | `0.04` | 偏移更大 |
| 色差强度 | `caShift = edge*0.012` | `0.012` | 色散更明显 |
| 扫描线深度 | `scanline*0.10` | `0.10` | 线条更黑 |
| 颗粒强度 | `noise()*0.12 + noise()*0.06` | `0.12, 0.06` | 更噪 |
| 辉光阈值 | `smoothstep(0.6, 1.0, brightness)` | `0.6` | 更低→更多发光 |
| 辉光强度 | `glow*0.08` | `0.08` | 更亮 |
| 暗角范围 | `vignette = smoothstep(0.35, 0.9, dist)` | `0.35, 0.9` | 暗角范围 |
| 反光强度 | `sheen*0.08` | `0.08` | 屏幕反光 |
| 暖色温 | `col *= vec3(1.02, 0.98, 0.94)` | `1.02, 0.98, 0.94` | 更暖/冷 |

### 速度参数（所有 `uTime` 系数）

| 效果 | 当前值 | 原快速值 |
|------|--------|----------|
| 水平抖动 | `uTime*0.25, 0.4` | `0.7, 1.3` |
| 垂直抖动 | `uTime*0.3` | `0.9` |
| 撕裂带 | `uTime*0.5, 0.6` | `1.5, 2.0` |
| 颗粒 | `uTime*0.4, 0.2` | `1.2, 0.6` |
| 亮度闪 | `uTime*0.15, 0.5` | `0.4, 1.5` |
| 色彩闪 | `uTime*0.2` | `0.6` |

## 6. 重要踩坑记录

1. **中文文件名**：GLSL `texture2D` 加载纹理时，Three.js TextureLoader 走 HTTP 请求。中文 + 空格文件名不会被浏览器自动编码，必须用 ASCII 文件名。

2. **加法 vs 乘法**：噪点/扫描线用加法叠加会直接盖住照片，应该用乘法让效果"融入"照片的亮暗变化中。

3. **强度 ≠ 速度**：用户说的"猛烈"有时是强度不够，有时是太快看不见。这是两个独立维度——先调强度到满意，再调速度到舒服。

4. **双层噪点**：单层噪点太规律。用两个不同频率的噪点相加：高频小振幅 + 低频大振幅，效果更自然有机。

5. **条件触发 vs 持续**：撕裂带和色彩闪断用 `step(random, threshold)` 条件触发，比持续效果更有"故障"的真实感。
